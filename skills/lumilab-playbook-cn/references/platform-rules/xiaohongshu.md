# 小红书 — 必做约束 + know-how

> 来源：white0dew/XiaohongshuSkills + 跨项目 30+ XHS skill 沉淀
> 这是 Content Agent 生成小红书内容时**必须读**的文件。

## 必做约束（硬规则，不靠 LLM 推断）

```
1. 标题 ≤ 38 字（中文按 2、英文/数字按 1，严格）
2. 图文必有图（无图不可发；纯文字只能发评论）
3. 图视频不可混用（一篇笔记要么全图要么纯视频）
4. 首图必须可读（大字标题 + 高对比度，视频也是封面规则）
5. 比例 3:4（图）或 9:16（视频）
6. 标签 3-10 个（少于 3 个流量低，多于 10 个会被识别为标签党）
7. 发布前必让用户确认最终标题、正文、图片
8. 不在正文放外链（小红书会降权）
9. 不用 24h 内同一账号发 ≥3 篇（限流）
```

## 标题写法

- **数字开头**："5 个 / 3 个步骤 / 28 天" 转化高
- **疑问钩**："为什么..." "怎么..."
- **反共识**："不要..." "别再..."
- **emoji 限 1-2 个**，放在首位或末位

## 正文结构（黄金 4 段）

```
1. 钩子段（1-2 行）— 引出痛点 / 反共识 / 数字结论
2. 价值段 — 1-3 个核心 takeaway
3. 细节段 — 具体操作 / 例子 / 数据
4. 互动段 — 提问 / 邀评 / 引私聊
```

## 配图

- 6-9 张最佳（曝光峰值）
- 第 1 张是封面（标题大字 + 主题图）
- 中间图可以是步骤 / 对比 / 数据图
- 最后一张可以是 CTA（关注 / 加微信 / 评论关键词）

## 违禁词 / 软违规

**绝对禁**：
- 任何"导流到外部平台"的词（微信号 / 公众号 / 抖音号 / 小破站）
- 医疗 / 金融保证性表述（"治愈" "稳赚" "百分百"）
- 极端形容词（"最" "唯一" "第一"）
- 涉政 / 涉黄 / 涉赌

**软违规（容易限流）**：
- 直接说"链接在评论"
- 频繁 @ 大 V
- 标题党与正文不符

- 私信引导："想要详细文档 → 评论扣 1"

## 2025–2026 规则更新

### 更新 1：笔记审核延迟 + 评论权重升级（2025 Q3）

- **发布时间**：2025-09，小红书创作者中心公告
- **来源**：小红书官方「啄木鸟」治理通报 / 创作者后台站内信
- **变化**：
  - 新账号首篇笔记审核延迟从 2-6h 拉长到 6-24h，「冷启动观察期」延长。
  - 评论权重大幅上调：前 30 分钟的「有效评论数 + 评论字数」直接决定是否进入二级流量池。一句话「沙发」「打卡」类评论被算法识别后会反向降权。
- **影响**：
  - 旧的「秒发秒爆」节奏失效，必须搭配评论组。
  - 评论质量比评论数量更重要。
- **应对建议**：
  - 发布前准备 5 条「真问题型」评论（≥ 15 字、含具体场景），首 30 分钟内由真实账号发出。
  - 新账号前 5 篇刻意避开正午 12:00 / 晚 21:00 的高峰，选择 10:30 / 15:30 等冷门时段降低被压审概率。

### 更新 2：账号分级（薯条号 / 创作号 / 商业号）+ 违禁词清单扩容（2026 Q1）

- **发布时间**：2026-02，「啄木鸟 4.0」治理升级
- **来源**：小红书《社区规范 2026 版》+ 蒲公英平台公告
- **变化**：
  - 账号分级落地：「薯条号 / 创作号 / 商业号」三档，未报备的带货笔记直接限流到 0 曝光。
  - 违禁词增量：「私信我」「扣 1」「同款链接」等历史灰区词进入硬违规清单；「AI 生成」需主动声明，否则算法降权。
- **影响**：
  - 评论区导流剧本基本失效，必须改为「关键词回复 + 蒲公英合作」组合。
  - 纯 AI 文本笔记（无真人视角 / 无第一人称细节）会被「AI 痕迹检测」判定为低质。
- **应对建议**：
  - 商业笔记走蒲公英报备；非商业笔记彻底去掉「扣 1 / 私信」字眼。
  - AI 辅助生成的文案必须叠加真实第一人称细节（时间、地点、人名、价格、对话），用 Mom Test 式具体过去描述代替通论。
