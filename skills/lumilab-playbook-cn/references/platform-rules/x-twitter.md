# X / Twitter — 必做约束 + know-how

## 必做约束

```
1. 首推 ≤ 280 字（含空格）
2. Thread 模式 2-7 条最佳（长 thread 流失大）
3. 数字 / 引语 / 反共识 开头
4. 单 emoji ≤ 2 个（多了像营销号）
5. Hashtag ≤ 2 个（X 算法不喜欢标签党）
6. 链接放最后一条 thread（首推有链接降权）
7. 不要 @ ≥3 个人（被识别为垃圾）
```

## 首推钩子（验证 5 种）

1. **数字**："I spent $0 on marketing. Here's how I got 10k users:"
2. **反共识**："Stop optimizing for X. Do Y instead."
3. **个人故事**："In 2024, I left my $200k job. 12 months later..."
4. **悬念**："The most underrated startup advice I've ever heard:"
5. **结论前置**："Founders ship features. Operators ship outcomes."

## Thread 结构（黄金 5 段）

```
1/ [钩子推] - 抓注意力
2/ [问题] - 用户为什么需要看下去
3/ [洞察 1] - 第一个 takeaway
4/ [洞察 2] - 第二个 takeaway
5/ [洞察 3 / 案例] - 具体例子
6/ [CTA] - "Follow me for more" 或 "Share if useful"
```

## 中文用户在 X 注意点

- X 上中英混发 OK，但**单 thread 要么全中要么全英**（混语会降低 engagement）
- 中文 thread 字符数计算：中文 = 1 字符（Twitter 改了规则），所以 280 字符约 = 280 个汉字
- 时差：中国晚上 8-10 点 = 美国早上 8-10 点（双高峰发推）

## Image 处理

- 推荐尺寸：1200×675（16:9）或 1080×1080
- 每条推 ≤ 4 张图
- 图片自带文字 OK，但要确保移动端可读

## 2025–2026 规则更新

### 更新 1：Premium 算法权重 + 长推文 vs 串推（2025 Q4）

- **发布时间**：2025-12，X 官方工程博客 + Elon 公开推文
- **来源**：X engineering blog「For You 排序权重 v3」+ @XEng 帐号说明
- **变化**：
  - Premium / Premium+ 订阅者的推文在 For You 流里获得固定加权，但加权额度从历史的 ~4x 降到 ~2x，且对「无互动的纯发帖号」失效。
  - 长推文（≥ 280 字，Premium 专属）的展开率 / 阅读完成率被纳入排序信号；表现差的长推文不再保留权重。
  - 串推（thread）权重持续高于单条长推文，因为「逐条点赞 / 转发」累计互动信号更密集。
- **影响**：
  - 仅靠订阅 Premium「买流量」的策略红利消退。
  - 长推文如果没人读完反而拖累账号长期权重。
- **应对建议**：
  - 4–7 条的串推 + 第一条「钩子句 + 数字承诺」仍是性价比最高的格式。
  - 长推文留给真正需要深度的话题（≥ 500 字 + 内嵌图片 / 数据），普通话题用串推。

### 更新 2：Community Notes 全球化 + Reply 排序变化（2026 Q2）

- **发布时间**：2026-04，X Community Notes 团队公告
- **来源**：communitynotes.x.com 更新日志 + Bloomberg 报道
- **变化**：
  - Community Notes 默认对所有语言开启，包括中文；被挂 Note 的推文曝光下降 ~80% 且原作者无法删除 Note。
  - Reply 排序从「时间 + 互动」混合切到「作者粉丝信任度 + 评论质量」加权，无认证 / 低粉账号的 reply 沉底，认证账号 + 高质量 reply 永远顶部。
- **影响**：
  - 任何带数据 / 引用的推文都可能被 Community Notes 介入，错误信息成本极高。
  - 评论区运营需要更高质量账号，而非数量。
- **应对建议**：
  - 引用外部数据时附原始来源链接（不要只放截图）；发布前对数字 / 时间 / 名字过一遍 fact-check。
  - 评论区准备「认证小号矩阵」或邀请同温层 KOL 评论，比让小号刷数量更有效。
